外部通知接口

通知返回的区别

returnUrl：支付完成后的同步跳转返回

支付完成后立刻返回到商户网站的客户端上，是可见的，返回机制：以 POST的方式返回，返回信息包括提交给我司的订单信息，可根据这个返回信息做相应的操作显示给客户看。参数获取方式例如：request("trade_status")

notifyUrl：支付完成后的异步通知返回

服务器的通知返回是由我司的服务器发起，以 POST 的方式返回到商户的网 站上。返回信息包括提交给我司的订单信息，在返回的文件中，注意需要输出**success**做为我司通知返回信息成功，若没有这个 success 的输出，那么我司的 服务器会在交易产生后的 24 小时内返回支付订单信息，返回的时间频率会逐渐 减弱，（1 分钟、3 分钟、5 分钟、10 分钟、15……），参数获取方式例如：request.Form("trade_status")
同步、异步通知参数列表

注意：

  1. 参数sign、signType不参与加密，其他有数的参数均参与加密，需注意参数值为0的参数，也需参与加密。
  2. 当支付渠道为支付宝（ALIPAY）、QQ（QQPAY）、网关时，异步通知返回扩展字段ext_param2，赋值为相应的渠道编码，如：ext_param2=ALIPAY等。
  3. 只有在跳转页面中输入正确登陆密码和支付密码后才能创建交易和通知返回，从集成后的系统健壮性考虑，收到我司发出的通知后，合作伙伴系统须判断接收到的交易状态是否与自己系统中的参数对应。如果不判断，存在潜在的风险，合作伙伴自行承担因此而产生的所有损失。

代付提交接口

请求参数

备注：手机号，证件类型，证件号，用户协议号可为空（为空的信息，位置必须保留）。

 

明细说明
  1. 将“序号,银行账户,开户名,开户行名称,分行,支行,公/私,金额,币种,省,市,手机号,证件类型,证件号,用户协议号,商户订单号,备注”按顺序排列，以英文逗号分隔。
  2. 如需使用支付宝代付，则“银行账户”输入支付宝登陆帐号（支持邮箱和手机），“开户名”输入支付宝用户姓名，“开户行名称”输入“支付宝”。
  3. 开户行名称需严格按照文档末尾的银行名称列表来传。
  4. 多条明细用|分割。
  5. 单笔金额最低为1元。
