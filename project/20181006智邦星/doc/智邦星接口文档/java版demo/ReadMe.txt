一、支付前准备
	支付前请向支付商获取必要参数，包括
	商户号(merchantCode)
	商户密钥(merchantKey)
	支付网关地址(payUrl)
	
二、支付过程
	1、支付网关接收POST请求
	2、将请求参数按ASCII顺序拼接成参数，并加密成密钥
	3、数据处理请参考demo中的代码
	
三、注意事项
	1、支付参数中,当前时间(currentTime) 为服务器当前时间(yyyyMMddHHmmss)，与网关时间误差不能超过30分钟，否则会拒绝处理　
	2、支付参数中,订单号(orderNo)不能重复
	3、不建议在同步回调中处理业务关系
	4、异步回调仅能返回(SUCCESS/ERROR)，其中，SUCCESS仅代表已接收数据，比如由于网络问题，支付方可能会多次请求，所以必须处理好可能出现的并发、重复调用的问题
